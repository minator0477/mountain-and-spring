## 1. 名山データの準備（緯度経度 ＋ 登頂記録）

名山の位置情報と登頂記録は CSV → GeoPackage 変換スクリプトで一括処理します。

### ディレクトリ構成

```
data/
├── original/
│   ├── public/           # 緯度経度・山名などの基本情報
│   │   ├── 100meizan02.csv
│   │   ├── 200meizan02.csv
│   │   └── 300meizan02.csv
│   └── private/          # 個人の登頂記録（Git 管理外）
│       └── meizan-record.csv
└── result/
    └── meizan.gpkg       # 変換後の出力ファイル
```

### 1-1. 基本情報 CSV の形式

`data/original/public/` 以下の 3 つの CSV は共通の列構成です。

| 列名 | 内容 | 例 |
|------|------|----|
| `No` | 通し番号（百名山 1〜100、二百名山 101〜200、三百名山 201〜300） | `1` |
| `山名` | 山の名称 | `利尻山（利尻富士）` |
| `北緯` | 緯度（十進法） | `45.1786111` |
| `東経` | 経度（十進法） | `141.2419444` |
| `よみがな` | 読み仮名 | `りしりざん` |
| `標高（m）` | 標高（整数） | `1721` |
| `所在地` | 都道府県など | `北海道` |
| `地域名` | 地域ブロック名 | `北海道` |
| `備考` | 補足情報（任意） | `H26年標高変更` |

### 1-2. 登頂記録 CSV の形式

`data/original/private/meizan-record.csv` に個人の登頂履歴を記録します。

| 列名 | 内容 | 例 |
|------|------|----|
| `記録ID` | 連番（任意） | `1` |
| `年月日` | 登頂日（`YYYY/MM/DD` 形式） | `2019/05/03` |
| `山頂ID` | 対象山の `No`（基本情報 CSV と一致させる） | `44` |
| `山名` | 山の名称（参照用・処理には不使用） | `筑波山` |
| `山行ID` | 山行単位の管理 ID（任意） | `1` |
| `備考` | メモ（任意） | `` |

同じ山に複数回登頂した場合は行を追加するだけです。

### 1-3. GeoPackage への変換

プロジェクトルートで以下を実行します。

```bash
# 登頂記録なし（位置情報のみ）
uv run python src/meizan_csv-to-geojson.py

# 登頂記録を統合（count / visits プロパティが付与される）
uv run python src/meizan_csv-to-geojson.py --with-records
```

実行後、`data/result/meizan.gpkg` が生成（または上書き）されます。
サーバー起動中でも反映には再起動が必要です。

## 2. 温泉情報の入力

温泉データは専用の管理 UI（`spring.html`）から CRUD 操作します。

### 2-1. 管理サーバーの起動

```bash
uv run python src/spring_server.py
```

ブラウザで `http://localhost:8001` を開きます。

> メインマップ（ポート 8000）とは独立したサーバーです。
> 同時に起動してもポートが競合しません。

### 2-2. 温泉の新規追加

1. 左サイドバーの **「＋ 新規追加」** をクリック
2. 以下の項目を入力

| 項目 | 必須 | 内容 |
|------|------|------|
| 名前 | ✓ | 温泉・施設名 |
| よみ | ✓ | ひらがなの読み |
| 泉質 | — | 複数選択可（単純温泉・硫黄泉 など 10 種） |
| 施設種別 | — | スパ／下町銭湯 |
| 座標 | ✓ | 下記いずれかの方法で設定 |
| 入湯記録 | — | 日付（`YYYY/MM/DD`）＋メモを複数行追加可 |

**座標の入力方法（どちらでも可）**

- **地図クリック** — フォームを開いた状態で地図上をクリックすると自動入力されます
- **テキスト入力** — Google マップで右クリックしてコピーできる `緯度, 経度` 形式（例：`35.676200, 139.650300`）をそのままペーストします

3. **「保存」** をクリック → `data/result/springs.gpkg` に書き込まれます

### 2-3. 既存データの編集・削除

- リスト or 地図上のドットをクリック → 編集フォームが開きます
- 内容を変更して **「保存」** で上書き
- **「🗑 このデータを削除」** で削除（確認ダイアログあり）

### 2-4. API エンドポイント（直接操作する場合）

| メソッド | パス | 内容 |
|----------|------|------|
| `GET` | `/api/springs` | 全件取得 |
| `POST` | `/api/springs` | 新規作成 |
| `GET` | `/api/springs/{id}` | ID で取得 |
| `PUT` | `/api/springs/{id}` | ID で更新 |
| `DELETE` | `/api/springs/{id}` | ID で削除 |
| `GET` | `/api/springs/name/{name}` | 名前で検索（部分一致） |
| `PUT` | `/api/springs/name/{name}` | 名前で更新（完全一致・一意） |

Swagger UI は `http://localhost:8001/api/docs` で確認できます。

## 3. Render でのデプロイ設定

プロジェクトルートに `Dockerfile` が含まれているため、**Docker** ランタイムを選択します。

### 3-1. サービスの作成

1. Render ダッシュボード → **「New +」** → **「Web Service」**
2. リポジトリを接続
3. 設定項目を以下のとおりに入力

| 項目 | 値 |
|------|----|
| **Runtime** | `Docker` |
| **Dockerfile Path** | `./Dockerfile`（デフォルトのまま） |
| **Build Command** | （空欄でよい。Dockerfile 内で完結） |
| **Start Command** | （空欄でよい。Dockerfile の `CMD` を使用） |

### 3-2. 環境変数

`APP_ENV=production` は Dockerfile 内で設定済みのため、追加設定は不要です。
必要に応じて以下を追加してください。

| 変数名 | 用途 |
|--------|------|
| `PORT` | Render がポートを上書きする場合（通常不要） |

### 3-3. データファイルについて

`data/result/meizan.gpkg` と `data/result/springs.gpkg` はリポジトリに含める必要があります。

- **`.gitignore` に追加している場合** は除外ルールを確認してください
- `.gpkg` はバイナリファイルですが Git での追跡が可能です
- Render はデプロイのたびにイメージを再ビルドするため、データを永続化したい場合は **Render Disk**（永続ストレージ）のマウントを検討してください

### 3-4. デプロイの流れ

```
git push origin main
  └─ Render が自動検知
       └─ docker build（uv sync + npm install + npm run build）
            └─ docker run（FastAPI が :8000 で起動）
```

デプロイ完了後、Render が発行する URL（例：`https://next-visit.onrender.com`）でアプリにアクセスできます。
